Python Coding Challenge for RA Applicants

Thank you for your interest in working with us!

In this challenge, you'll carry out some operations on mock numerical data. The operations you'll need to implement are similar to steps you might encounter in a functional magnetic resonance imaging (fMRI) analysis pipeline. You should use the Python programming language (with numpy, scipy, & matplotlib) to solve the problems. Please produce one script that generates all of the requested results figures. While it is important that your script give accurate results, applicants will be assessed most strongly on the implementation (e.g., well-thought out, well-commented, readable, organized).

If you have any clarification questions about the instructions, please send them to sr1209@georgetown.edu. We will not be able to answer any conceptual questions about the material (e.g., Python, image processing, fMRI, cognitive neuroscience). Therefore, we encourage you to make use of publicly available resources, such as Google, YouTube, online forum archives (e.g. Stack Overflow), and software documentation. Be sure to document with an in-line comment anytime a code snippet comes from an external resource. However, you should not directly recruit any external help, e.g. from friends or online help forums. This is meant to be a test of your own problem solving skills.

You will have 1 week to complete as much of the test as you can. When you are finished, please upload your script + results figures as a .zip file named with the heuristic "LASTNAME_FIRSTNAME.zip" to the link below. The problems are intended to be challenging, and we do not necessarily expect you to complete all of the problems perfectly. Feel free to view example outputs at this link. Please comment your thought-process throughout the script. Good luck!

Section 1. Image Processing

In this section, you will read a grayscale JPEG image (please download otter.jpg here) into Python as a numerical matrix and carry out some operations on it.

  1. Plot a histogram. Plot a histogram of the image intensities in the JPEG. Include 10 histogram bins. Save the histogram plot as either a pdf or png file. (e.g., otter_hist.pdf)
  2. Scramble the image. Shuffle the pixels in the image (without altering the values) and save the scrambled image as another JPEG. (e.g., otter_shuffled.jpg)
  3. Spatially smooth the image. Smooth the image by convolving with a 2D Gaussian kernel. The kernel should have a full-width at half-maximum (FWHM) of 10 pixels. Save the smoothed image as another JPEG. (e.g., otter_smoothed.jpg)

Section 2. Simulate a BOLD time-course

In this section you will create a simulated blood-oxygenation-level dependent (BOLD) time-course. The time-course will be based on a predicted neural response, following an on-off stimulus presentation. The relationship between BOLD signal and neural firing is modeled as a linear time-invariant system. A hemodynamic response function (HRF) represents the predicted BOLD signal following a brief neural impulse. The predicted BOLD signal in response to a train of neural firing is obtained by convolving thetime-course of neural firing rates with the HRF. Here you will create such a BOLD time-course, starting with a neural firing time-course, and using a standard HRF (hrf.txt).

  1. Plot the HRF. Read the HRF text file (please download hrf.txt here), which contains a column of small numbers, into Python as a numerical vector. Plot the HRF as a continuous line and save the figure as either a pdf or png file (e.g., hrf.pdf). Importantly, the HRF is in units of 2 secs. It contains 16 values, and so represents a time window of ~32 seconds. Notice the slow rise and fall of the HRF, which is one of the reasons for the lack of temporal precision in fMRI.
  2. Generate a predicted neural time-course. Generate a predicted neural time-course based on an on-off stimulus presentation paradigm. The time-course should be 200 secs long, but have the same temporal units as the HRF (2 secs). Model the neural time-course as a "boxcar" function, with stimulus "on" periods taking a value of 1.0, and stimulus "off" periods taking a value of 0.0. On and off periods should be 20 secs long. Plot the predicted neural time-course and save as a pdf or png. (e.g., on_off.pdf)
  3. Convolve neural time-course with the HRF. Convolve the simulated neural time-course with the HRF, and plot them together (both 200 secs long). Save figure as a pdf or png. (e.g., on_off_convolved.pdf)

Bonus:

  • Add low frequency noise. fMRI data is often plagued by low-frequency noise. Add low-frequency noise to your convolved BOLD time-course, in the form of a sine wave with a period of 300 secs, and amplitude 0.3. Plot and save a figure. (e.g., convolved_drift.pdf).
  • Add white noise. Finally, add random Gaussian noise at each time point (use sigma=.2). Plot and save a figure. (e.g., convolved_drift_noisy.pdf)
